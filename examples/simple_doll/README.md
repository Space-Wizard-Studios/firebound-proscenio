# simple_doll fixture (SPEC 007)

Tests the **`1 sprite = 1 PNG`** workflow — the common case for the
Photoshop-first pipeline. Each sprite mesh references its own
dedicated PNG; the addon's Pack Atlas operator is expected to produce
a packed atlas with the 5 sources side-by-side, **no slicing required**.

## Contents (after running the build script)

```
simple_doll/
├── layers/
│   ├── head.png          64x64
│   ├── torso.png         96x128
│   ├── arm.L.png         32x96
│   ├── arm.R.png         32x96
│   └── legs.png          80x96
├── simple_doll.blend
├── simple_doll.expected.proscenio    golden — CI diffs against re-export
├── Doll.tscn                          Godot wrapper (manual user pattern, SPEC 001)
└── Doll.gd                            empty stub
```

## Skeleton

```
root
├── spine
│   ├── head            (head sprite parented + weighted to this bone)
│   ├── arm.L           (arm.L sprite weighted 1.0 here, 0.3 to spine — multi-bone weights)
│   └── arm.R           (mirror)
└── legs                (legs sprite weighted 1.0 here)
```

`torso` is parented to `spine`. Multi-bone weights on the arms exercise the SPEC 003 weight-export path.

## Actions

- `idle` — 4-frame loop, tiny vertical bob on `spine`.
- `wave` — 8-frame, animates `arm.R` rotation 0° → -90° → 0°.

## Building from source

The fixture is generated by a deterministic headless-Blender script:

```bash
blender --background --python scripts/fixtures/build_simple_doll.py
```

Output is committed to the repo. Re-run whenever the fixture spec changes.

## What this fixture catches when broken

- Pack Atlas regression for `1 sprite = 1 PNG` (slicing logic should be a no-op pass-through here).
- Writer regression on multi-bone weights.
- Action export regression (two actions, one with rotation tracks).
- Schema bumps that break the golden `.proscenio` diff.
