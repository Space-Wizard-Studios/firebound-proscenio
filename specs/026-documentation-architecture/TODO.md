## Wave 2: comment cleanup execution (Axis B), by category

Consume the Wave 0 worklist. Stage as small per-area PRs (one app or one domain at a time), not one sweep, to keep diffs reviewable and avoid conflicts. Order within the wave, lowest risk first:

- [ ] delete: echo, padding, refactor archaeology, and every spec-path reference comment and docstring.
- [ ] tighten-inline and to-assert: reduce a local invariant to its constraint, or promote it to an enforced `assert` / `raise`.
- [ ] to-module-docstring: move module-purpose rationale to the module docstring, only where the non-obvious bar is met.
- [ ] to-test-link: convert regression rationale to a named regression test plus a terse pointer comment. Gated on the guarding test existing; where it does not, log the missing test rather than deleting the knowledge.
- [ ] to-architecture: land the few cross-cutting items (XZ picture-plane convention, golden field-order test constraint) in architecture.md's "Where to tread carefully" section.

Spans `.py`, `.gd`, `.ts`. Generated files are never touched.

Gate per PR: the affected app's full test + lint gates pass (Blender: `uv run pytest` from the app dir per the blender test gates, ruff, mypy; Godot: gdformat + gdlint; Photoshop: tsc + vitest). No behavior change; diffs are comment-only except the to-test-link tests and to-assert promotions.
